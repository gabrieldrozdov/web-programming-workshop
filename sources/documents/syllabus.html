About the Workshop

Syllabus!

1. What is this?
1.1  All About the Workshop

The Web Programming Workshop is a few things:

  1. A curriculum of coding lessons and projects centering on design;
  2. A resource for learning about and referring back to coding concepts; and,
  3. A tool for learning, writing, and teaching code.
1.2  Course Description

Code is intimidating. To start coding, it can feel that you need to learn everything before creating anything of value. Web programming offers an alternative path by proving that you don’t need to know much about code to make incredible things with it. In this course, we’ll explore the field of designing with code through the lens of web development, discovering the capabilities of coded experiences and crafting projects that explore those possibilities. By working directly with HTML, CSS, and JavaScript, you’ll gain an understanding of how code fundamentally changes the design process and see firsthand the exciting outcomes that emerge.

2. How does this course support inclusivity?
2.1  Accessibility

The Web Programming Workshop is part of GD with GD, an open-source educational initiative aiming to bring high quality design instruction to as wide of an audience as possible. This workshop specifically tackles the field of web design through the lens of web development. While there are numerous free online educational resources for learning code, there are very few that target individuals who don’t want to become full-time developers but still want to work with code. The Web Programming Workshop’s target audiences are designers, artists, and other creatives who want to use code in their work but don’t need or care about computer science principles like optimization or scalability. These individuals want to create art with code and may be intimidated and deterred by traditional coding curricula.

Educational institutions have track records of excluding or neglecting students of various identities and backgrounds. Diversity and inclusivity statements are a small-but-important effort at reckoning with these histories and striving to do better. In this course, I hope to create a space in which all students are supported, regardless of their gender, sexual orientation, abilities, age, body size, socioeconomic circumstances, ethnicity, race, culture, religion, and anything I am forgetting. You are accepted, valued, and respected for exactly who you are, and I encourage and trust you to feel the same for your peers.

If you need accommodations to complete this course’s requirements, please reach out as early as possible. We’ll work together with Disability Support Services to make sure your needs are met and that you can succeed in this class.

3. What is the point of this course?
3.1  Goals

These are the conceptual objectives we’ll tackle in this course:

  • To grasp how computers use code to create websites.
  • To develop interactive web experiences by combining the functionalities of HTML, CSS, and JavaScript.
  • To establish a workflow for writing code, referencing online resources, and testing websites both locally and online.
  • To understand what makes a design easy or hard to program.
  • To uncover new design possibilities unique to websites
3.2  Outcomes

These are the tangible skills you’ll leave the course with:

  • Foundational HTML/CSS programming skills and experience.
  • Versatility with reading, integrating, and editing pre-existing Javascript code.
  • Ability to plan and write new JavaScript code.
  • Analytical skills for determining and resolving bugs in code.
  • Research skills for solving technical challenges, including understanding and utilizing reference documentation.
  • Vocabulary to communicate with web programmers.
  • Exposure to design and programming tools for digital platforms.
4. What will we be doing?
4.1  Structure

This course is split between lessons, exercises, and projects.

Lessons follow a curriculum that builds up coding skills starting from scratch all the way to generating web pages via data from JSON files. While lesson lengths vary, each one is meant as a bitesize portion focusing on a specific topic.

Contained within lessons are live demos that introduce new concepts, which are then explored further during in-class exercises. These exercises are focused explorations showcasing the creative possibilities of the topics covered in the relevant lesson.

Projects offer open-ended opportunities for working with code and trying out concepts covered during lessons and exercises. These projects may present challenges or restrictions that force clever solutions to coding prompts.

4.2  Materials

Besides the class website, the course uses the following materials:

  • Visual Studio (VS) Code for creating and modifying HTML, CSS, and JavaScript files.
  • A GitHub account, GitHub Desktop, and GitHub Pages for publishing and launching web projects.
  • Google Drive for submitting assignments.

Check out the Resources document for additional

4.3  Critiques

Our critiques will focus on two qualities: 1) technical proficiency, and 2) creative exploration. In web programming, these qualities go hand-in-hand and will either support or diminish each other. Your goal should be to address any major technical concerns before a critique. During the critique, we can talk about ways to take the work even further. The expectation is for you to understand exactly what your code is doing, so be ready to explain your code!

5. How will I be graded?
5.1  Requirements and Expectations

Since this course is a technical workshop, I have decided to grade it on a rubric. This means that your grade exactly corresponds to the work you complete as part of the class.

See “Assessment” below to calculate your own grade for the course.

You can also take this course pass/fail if you’d like. To pass the course, you must attend all class meetings and submit both assignments.

5.3  Participation and Attendance

Attendance is required for all weekly meetings. If you have an excuse for missing a class, you must let me know at least 24 hours in advance. An unexcused absence is a direct reduction of your grade.

If you miss a class, you are also responsible for catching up on the class’s lessons and completing any relevant exercises. These materials will always be available online at the class website.

5.4  Plagiarism

Plagiarism for code is a tricky subject. You’ll likely be copying code from a variety of online sources (including this site), but that doesn’t necessarily mean you’re plagiarizing.

Here are a few things you should be wary of:

  • Using code that you don’t understand or can’t explain.
  • Turning in an assignment with largely the same code as a peer or former class member.
  • Copying an entire webpage and turning it in as your own work.
  • Exclusively copying code from other sources instead of attempting to write code from scratch.

If you are struggling with writing your own code and have to rely exclusively on other sources, please don’t hesitate to reach out to me for additional help or make an appointment for office hours. There’s no point in taking a coding course if you aren’t going to learn how to code, so please find whatever support you need to succeed!

6. What is our schedule?
6.1  4-week Workshop

This format of the course offers the basics of web programming in a relaxed introductory schedule. Instruction focuses mainly on HTML and CSS, with some JavaScript if time permits and the cohort seems comfortable with more advanced topics. Grading occurs on a rubric and encourages creative experimentation more than technical proficiency.

6.2  5-week Intensive

This format of the course crams 12 full meetings in the span of 5 weeks. The schedule is based on RISD’s wintersession structure, which features semi-regular meeting times on consecutive days each week. Since this structure doesn’t offer students much time to explore code outside of class meetings, the curriculum is slightly abbreviated and adaptable based on the comfort level of the cohort.

6.3  12-week Semester

This format is the standard course structure of 12 consecutive weeks with a consistent meeting time each week. The schedule covers the core curriculum in its entirety, and students are given ample time to both explore the possibilities of code as well as accumulate best practices along the way.
