Things to Learn From
Documentation

Web documentation is like a dictionary for a language you can’t speak. Even as you get better at coding, you’ll always want a place to refer back to in case you forget or misremember something.

These websites contain every HTML tag, CSS property, JavaScript function, and more that you could ever write. As you learn to code, you’ll also learn how to quickly reference web documentation since there’s no expectation that you have everything web-related memorized.

Here are some popular web documentation resources:

  • My current go-to for everything is MDN Web Docs. It is not always beginner-friendly, but it usually contains the most information out of all web documentation sites. Also, MDN is run by Mozilla and is more of an “official” web documentation as compared to other sites.
  • My previous go-to for everything was W3Schools. It contains tons of helpful demos and is generally more beginner-friendly, but isn’t always up-to-date on best practices.
Flexbox and Grid

The CSS properties “flex” and “grid” are essential for organizing and laying out content. They can be a bit confusing though, so many people have developed resources to make them easier to understand.

  • If you’re really struggling, I highly recommend Flexbox Froggy and Grid Garden. These are two games that teach you the basics of flexbox and CSS grid.
  • If you’re feeling more comfortable with the basics, then you can use the “complete guides” to Flexbox and CSS Grid as cheat sheets. These websites are overviews of all the related CSS properties and include explanatory visuals.
Code Examples/Snippets

Sometimes you need an example of code in action to learn how it works. There are tons of resources for finding and working with code snippets, but be careful! You want to avoid creating a monstrosity of copy-pasted code that you don’t understand (plus, that’s plagiarism!). That being said, there’s nothing wrong with learning how to code something from an example as long as you can explain what your code is doing and rewrite it from scratch. This is how I continue to learn!

Here are a few resources that’ll help you out if you’re looking for snippets of code:

  • For code snippets, check out CodePen. This is your best resource for searching for a specific thing and finding one or multiple direct examples.
  • For Q&A-style problems, check out Stack Overflow. You’ll often come across Stack Overflow pages on Google if you’re searching for a specific question.
  • For clever tips and techniques, check out CSS-Tricks. I think of this site as a resource for discovering solutions to problems you didn’t even know you had.
YouTube Channels

There are hundreds of free tutorials on YouTube (and elsewhere) that you can pick-and-choose from based on what you’re specifically trying to learn. To start, I’d recommend plugging in a basic search (like this) and looking for a video/series that has a lot of views and is relatively recent (1–2 years old max).

Here are a few YouTube channels I’m semi-familiar with, but note that these are not necessarily the best ones out there:

  • For tutorials spanning a whole suite of web programming practices ranging from basic to industry-standard, check out Syntax (previously LevelUpTuts) and Net Ninja.
  • For tutorials focusing on the more creative side of coding, particularly the JavaScript library p5.js, check out The Coding Train.
  • For an alternative set of tutorials, check out Webflow, which is a website builder whose tutorials are top notch and can help you hand-code websites too.
Things to Use
Working with Code

There are countless ways of working with code and text documents, but this is what I’ve settled on for my work:

  • Visual Studio (VS) Code for editing text documents.
  • GitHub Desktop for managing code repositories.
  • GitHub Pages for launching repositories as static websites.
VS Code Extensions

You can increase VS Code’s functionality by installing extensions. Here are some that I recommend:

  • Live Server for locally testing websites (with automatically refreshing pages when you save your file!).
  • Bookmarks for jumping to specific spots in your code.
  • intent-rainbow for keeping your code cleanly formatted.
  • Select Lines Count to easily see how many lines of code you have highlighted..
  • This isn’t an extension, but I also recommend turning on line-wrapping in VS Code so that you don’t have to horizontally scroll to see what you’re doing.
Web Browsers

When you design a website, you design it for everyone, not just for everyone using your web browser. So, you need to test your website in multiple browsers (and at multiple screen sizes) to make sure it runs properly.

The good news is that there are only three web browsers you really need to care about:

  • Google Chrome
  • Mozilla Firefox
  • Apple Safari
Browser Extensions

I primarily use Chrome, and in Chrome I have a few extensions I use regularly for testing and analyzing websites.

Here are my favorites:

  • WhatFont for viewing the fonts used on a website.
  • Wappalyzer for viewing what a website is built on top of.
  • JSONView for making JSON files more legible directly in Chrome.
  • GoFullPage for capturing a screenshot of the entire website (including scrollable regions).
  • Scroll Capture for capturing a video of the website with keyframed controls.
Fonts

In any kind of visual or graphic design, the typefaces and fonts you use are incredibly important. The same is true for web programming and design. Thankfully, we can use much of the same fonts we use in other mediums, with the stipulation that we actually have the font files available to us.

Here are a few places to source fonts (legally) from:

  • WhatFont lets you use a massive library of premium fonts, although you have to load the files via their servers. This process is easy enough, but keep in mind that you have to have an Adobe subscription to keep these fonts active.
  • Google Fonts is the largest repository of free fonts you’ll find. They also have an option to load in fonts via their servers.
  • “Web-safe” fonts are fonts that you can use freely since most computers already have them installed. This varies per computer though, so make sure to check the availability of a font before attempting to use it.
  • There are several open-source font foundries dedicated to releasing free fonts. A few of note are Velvetyne, The League of Moveable Type, UNCUT, Collletttivo, and Open Foundry.
  • To find more free fonts, I recommend searching through collections on Are.na that other people have made.
Site Builders

Coding websites by hand is exciting and limitless. There is one catch, though — real-world websites need to be maintained, and most in-house teams can’t (or don’t want to) work with code directly. They want a content management system (CMS) that lets them easily add content or build new pages. Luckily, there is a whole industry of site builders that include built-in CMS.

Here are some you may encounter or want to try out:

  • My personal favorite is Webflow, which functions a bit like an Adobe app for websites. With Webflow, you can precisely design your website much like you would if you were coding it by hand. The downside is that it’s harder for clients to design their own pages directly, but Webflow includes a customizable CMS to help in this process.
  • Squarespace, Wix, and Cargo are all similar site builders that feature relatively straightforward drag-and-drop interfaces. These are great for projects that don’t require too much customization, particularly in cases when a client wants to create their own pages later on without the help of a designer.
  • WordPress.org (not WordPress.com!) runs most of the internet. It’s a little harder to get set up because it runs more like an operating system than a site builder and has to be installed manually on a web server. The upside to WordPress is its breadth, featuring thousands of extensions and themes that let you precisely customize a website’s look and functionality. The downside is also this breadth — the other site builders exist because of WordPress’s complexity. Nevertheless, there are ways to make WordPress extremely user-friendly with similar (and often more in-depth) drag-and-drop page builders.
Domain Names (URLs)

Websites are primarily made up of two parts — a server that hosts your website’s actual code, and a domain name (i.e. URL) that lets you access that code. The server and domain name are often purchased independently and both run on subscription models (a typical domain is around $12–18/yr).

Here are some places you can buy a domain name subscription from:

  • A great, reliable provider for domain names is Namecheap. Subscription rates are industry-standard, and the site is easy to use with no extra frills.
  • If you end up using a site builder, they’ll also often offer promotions to include a free domain with your web hosting subscription plan (i.e. how much they charge you for their service). You should just make sure to check what the renewal rate is for your domain, as well as if it’s easy to link your domain to another website in the future if you want to move your project off of the site builder..
JavaScript Libraries

Once you have a hang on HTML, CSS, and the basics of JavaScript, you can discover exciting new possibilities with the use of JavaScript libraries. These libraries extend the functionality of your website by providing you with new JavaScript functions.

Here are a few fun ones to check out:

  • For creatively coding and generating graphics, check out p5.js.
  • For using machine learning functionality like image recognition, check out ml5.js.
  • For working with audio sampling and synthesis, check out Tone.js.
  • For manipulating vector paths, check out Paper.js.
  • For working with 3D models, check out Three.js.
  • For physics-based simulations, check out Matter.js.
  • For running After Effects vector animations natively on the web, check out Lottie.
